OAuth (Open Standard Authorization)
    介紹
        使用者允許該應用程式有限度取得 Facebook、Google 等平台的相關資料，例如:登入畫面上的 FB 圖示 或 Google 圖示
    讓系統組件獲取另一個系統組件的訪問權限
        1.資源擁有者
            指使用者本身或系統中的帳號，利用萬能鑰匙產生出有限鑰匙轉交給他人的訪問授權過程
        2.某種受保護資源
            OAuth 不會限制資料的格式，所以資料可以是 RESTfule API、某個網頁、URL 等種類資源
        3.客戶端
            資源擁有者訪問受保護資源的軟體，就是將鑰匙交給他人
    OAuth 不是?
        1.身分驗證協議
            OAuth 會注重在於"授權"上，而不是驗證
        2.用戶對用戶授權機制
            OAuth 有定義授權框架與流程，但不定義授權內容，會由服務 API 定義使用手權範圍，指定權杖的操作
        3.定義權杖格式
            OAuth 申明權杖內容對於客戶端完全不透明，權杖本身對於客戶端是不透明，但接受權杖處理的服務，仍然需要理解權限
        4.沒有定義加密方法
            為了安全，諸多服務如 Google、Spotify 都會客戶端支援 HTTPS，但框架不關注這部分，不管權限簽名、加密、驗證等
        5.單體協議
            OAuth 2.0 被規範成多個定義和流程，每個定義都有適用場景
